Task 1

Граф має такі характеристики:
1) Кількість вершин - 6
2) Кількість ребер - 9
3) Ступінь вершин :
   'Київ' : 5, 
   'Житомир' : 2, 
   'Вінниця' : 3,
   'Черкаси' : 3,
   'Полтава' : 3,
   'Чернігів' : 2


Task 2

Кількість кроків для знаходження шляху до всіх вершин графу з вершини 'Київ' наступна:  

   Алгоритм DFS рекурсивний: 6

   Алгоритм DFS ітеративний: 19

   Алгоритм BFS рекурсивний: 10

   Алгоритм BFS ітеративний: 10

   Можемо зробити висновок: з цим конкретним графом найшвидше спрацював алгоритм DFS рекурсивний. Однак, як відомо, DFS не гарантує, що перший знайдений шлях буде найкоротшим, на відміну від BFS, який вимагає більше пам'яті (що в нашому випадку теж видно), але гарантує знаходження найкоротшого шляху. В теорії для дуже глибоких графів DFS може бути ефективнішим за пам'яттю, тоді як для широких графів BFS може використовувати багато пам'яті. Тому необхідно вбирати варіант згідно поставленої задачі і структури графа.
